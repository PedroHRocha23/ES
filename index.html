Sobre a Engenharia de Requisitos, considere as afirmativas a seguir.

I. A Engenharia de Requisitos precisa passar por adaptações necessárias nos aspectos de processo, projeto, produto e de pessoal que está fazendo o trabalho, como todas as outras atividades de Engenharia de Software

II. A etapa de validação de requisitos tem como principal objetivo mostrar que os requisitos levantados realmente refletem a realidade do software que o cliente deseja. A ocorrência de erros no documento de validação de requisitos pode levar a grandes custos porque gera retrabalho.

III. Os requisitos levantados ficam mais consistentes, à medida em que a informação de vários pontos de vista é coletada.

IV. Na etapa de levantamento de requisitos, a equipe técnica de desenvolvimento do software trabalha em colaboração com o cliente e os usuários finais do sistema para descobrir que serviços o sistema deve oferecer, entender melhor o domínio da aplicação, o desempenho esperado do sistema, além das restrições de hardware, entre outras informações.

Qual é a melhor prática quando se decide Refatorar um trecho de código segundo a metodologia XP?

a. Fazer um plano de ação orientado por um líder técnico

b. Começar construindo os testes antes e depois ir fazendo pequenas modificações no código e ir testando após cada modificação.

c. Deixar o desenvolvedor mais experiente fazer os passos mais importantes

d. Designar um time separado do projeto para planejar o processo de refatoração
